Popconfirm 确认弹窗 ​

用于在执行存在风险或不可逆操作前提供二次确认的浮层组件，通常包裹触发元素（如按钮）。

基本用法 ​

将需要确认的触发器作为默认插槽放入 xr-popconfirm，在确认后触发 confirm 事件：

基本用法

点击按钮触发确认弹窗。

<template>
    <div class="popconfirm-demo">
        <h3>基本用法</h3>
        <p>点击按钮触发确认弹窗。</p>

        <div class="popconfirm-buttons">
            <xr-popconfirm title="确定删除吗？" @confirm="handleDelete">
                <xr-button type="danger">删除</xr-button>
            </xr-popconfirm>
        </div>
    </div>
</template>

<script setup lang="ts">
import { XrButton, XrPopconfirm, XrMessage } from 'xier-element'

const handleDelete = () => {
    XrMessage.success('删除成功')
}
</script>

<style scoped>
.popconfirm-demo {
    padding: 20px;
}

.popconfirm-buttons {
    margin-top: 10px;
}
</style>

自定义按钮 ​

自定义确认和取消按钮的文本和类型：

自定义按钮

自定义确认和取消按钮的文本和类型。

<template>
    <div class="popconfirm-demo">
        <h3>自定义按钮</h3>
        <p>自定义确认和取消按钮的文本和类型。</p>

        <div class="popconfirm-buttons">
            <xr-popconfirm title="确定要执行此操作吗？" confirmButtonText="确定执行" cancelButtonText="再想想"
                confirmButtonType="success" cancelButtonType="info" @confirm="handleConfirm" @cancel="handleCancel">
                <xr-button type="primary">执行操作</xr-button>
            </xr-popconfirm>
        </div>
    </div>
</template>

<script setup lang="ts">
import { XrButton, XrPopconfirm, XrMessage } from 'xier-element'

const handleConfirm = () => {
    XrMessage.success('操作已执行')
}

const handleCancel = () => {
    XrMessage.info('操作已取消')
}
</script>

<style scoped>
.popconfirm-demo {
    padding: 20px;
}

.popconfirm-buttons {
    margin-top: 10px;
}
</style>

禁用状态 ​

设置 disabled 属性可以禁用弹窗触发：

禁用状态

设置 disabled 属性可以禁用弹窗触发。

<template>
    <div class="popconfirm-demo">
        <h3>禁用状态</h3>
        <p>设置 disabled 属性可以禁用弹窗触发。</p>

        <div class="popconfirm-buttons">
            <xr-popconfirm title="确定删除吗？" disabled @confirm="handleDelete">
                <xr-button type="danger" disabled>删除（禁用）</xr-button>
            </xr-popconfirm>

            <xr-popconfirm title="确定保存吗？" @confirm="handleSave" style="margin-left: 10px;">
                <xr-button type="primary">保存（启用）</xr-button>
            </xr-popconfirm>
        </div>
    </div>
</template>

<script setup lang="ts">
import { XrButton, XrPopconfirm, XrMessage } from 'xier-element'

const handleDelete = () => {
    XrMessage.error('删除操作')
}

const handleSave = () => {
    XrMessage.success('保存成功')
}
</script>

<style scoped>
.popconfirm-demo {
    padding: 20px;
}

.popconfirm-buttons {
    margin-top: 10px;
}
</style>

API ​

Props ​

名称说明类型默认
title弹窗标题文本string-
confirmButtonText确认按钮文本string"确认"
cancelButtonText取消按钮文本string"取消"
confirmButtonType确认按钮类型ButtonType"primary"
cancelButtonType取消按钮类型ButtonType-
icon左侧图标（支持类名或组件）string"question-circle"
iconColor图标颜色string"#f90"
hideIcon是否隐藏图标booleanfalse
disabled是否禁用触发器booleanfalse
hideAfter自动隐藏延迟（ms），确认后自动隐藏number200
width弹窗宽度，支持数字或字符串（例如 320 或 320px）number | string150

Events ​

事件名说明参数
confirm点击确认后触发-
cancel点击取消后触发-
show弹窗显示时触发-
hide弹窗隐藏时触发-

Methods ​

方法名说明
show()手动显示弹窗
hide()手动隐藏弹窗

说明 ​

  • xr-popconfirm 通常包裹触发节点（如按钮、链接等）。
  • 可通过 Props 自定义按钮文本和样式，以适配不同场景下的确认交互。
  • 基于 Tooltip 组件实现，支持所有 Tooltip 的触发方式。
